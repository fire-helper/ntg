# Part 1

What do Chrome, or to be more precise, Chromium, Firefox and Android
have in common?  They are all *open-source software*.

Open-source software powers the world. As an end user, you probably
haven't heard about the countless lines of code underpinning modern
computing.

Just about 30 years ago, you couldn't even write code for free. Then, GCC
came into the world as free and open-source software, enabling millions
of people to start writing code for free. In 2026, there is no more paid
programming languages.

Just about 30 years ago, there were no free operating systems. They all
cost money to use. Then, Linux and the BSDs came into the world. Now,
the vast, vast, vast majority of the Internet runs on Linux, especially on
servers. *Every single* supercomputer in the world runs on Linux. Android
is the most popular operating system in the entire world, and it's
open-source.

Tools like Docker and Kubernetes define the "cloud". They allow developers
to coordinate thousands of computers and servers together. They are free
and open-source.

Apache and NGINX pioneered the Web, and were one of the first ever web
servers. They implemented many Web standards and came to be some of the
most popular Web software in the world. They are free open-source.

Most important of all is the unimaginably vast amount of FOSS
libraries. cURL and sqlite are *the* most deployed code in the
entire world and powers all of the Internet. They are free and
open-source. Chromium, the most popular web browser in the world, is
free and open-source.

# Part 2

But what really is "open-source"? According to the Open Source Initiative,
open-source software is defined by a few things:

- The source code must be available for anyone
- The code must be freely redistributable
- Allowance for derived works
- No discrimination against who gets to use the software

Open-source software is developed in the open: anyone gets to see the
code, contribute and fix bugs themselves.  Don't like something about
a program? Change it yourself.

Due to their nature, open-source software is a collaborative effort. Thus,
they have many eyes on them from many people. It is difficult to sneak
malware or malicious code into open-source software.

Contrasting this with proprietary software: everything is a black box. You
can't get to see how it works, you can't read the code. People can't
contribute to it, can't redistribute it, can't change it. A company can
put malware into proprietary software and you won't know it!

# Part 3

The benefits of open-source software cannot be understated.

On the technical side of things, software having the source code available
means anyone can actually verify and audit the code to make sure there
are no security problems or malicious code. Due to being open, it also
allows other software to inter-operate and work together.

Open-source software is typically also free of charge. This means
a large company can use them without having to pay a license cost for
every employee. This also greatly benefits students and individuals who
don't have to pay for costly software to learn.

All of this ultimately means that you, as the end user, is guaranteed
freedom: freedom to use, freedom to modify, freedom to distribute
software. You don't have to worry about an app suddenly going in a bad
direction: everyone can change it back themselves.

# Part 4

Copyright laws automatically assign copyrights to anything you create,
including code. By default, copyright means that no one can *use* your
code because that constitutes copyright infringement. To allow people
to use your code, you have to grant them *licenses*.

A license is basically an agreement that says you allow other people
who has a license to use your code. The easiest and most common method
is to distribute a license *along* with the source code, meaning anyone
who obtains the source code also automatically obtains a license.

Software licenses are largely divided into 2 categories: permissive
and copyleft.

Permissive licenses include those such as MIT, ISC, BSD, etc. MIT is by
far the most popular license in the world.  Permissive licenses allow a
lot of freedom to licensees. They can do just about anything they want
with your code.

Copyleft licenses include those such as the GPL family of
licenses. Copyleft requires modifications to the original code to also
be under the same license. This means no one can take code, modify it,
then make their version proprietary. Furthermore, if you use copyleft
code in a bigger project, the entire project must also be copylefted.

There are licenses that sit between permissive and copyleft, such as
the MPL.

# Part 5

You don't have to be a programmer to contribute to open-source. There
are many things anyone can do: fixing typos, writing documentation,
filing bug reports, test for bugs, helping other people out by answering
questions, etc. Most importantly of all, if you find the software useful,
consider donating money.

# Part 6

Many important projects are kept alive by a few volunteers or one
person. Funding is uneven — donations and corporate sponsorships exist
but aren’t always stable. Maintainers can burn out from constant
issue triage and patching. Small, widely used libraries can become
supply‑chain risks if they’re unmaintained or compromised.

Even worse, companies would take these free software, use them and give
0 cents back to the author.

Ways to improve sustainability include paid maintainer positions,
nonprofit foundations, commercial support, grant programs, and platforms
for sponsorship. Technical steps—like reducing unnecessary dependencies,
pinning versions, reproducible builds, automated tests, and security
audits—also reduce risk.

# Conclusion

Open source means you can see, change, and share code under rules set by
a license. It helps innovation, lowers cost, and builds trust through
transparency. But many projects need more stable funding and help to
stay secure and well maintained.

Pick one open‑source project you use this week and do one useful thing:
file a clear bug report, improve a piece of documentation, or submit
a small fix. If you run systems, audit your critical dependencies and
consider sponsoring maintainers. Small, regular actions from many people
make the whole ecosystem healthier. Don't know if you are using any
open-source software? Look through your list of apps, you definitely
are using some, you just didn't know it.
